# MLOps Course Project

An educational end-to-end MLOps pipeline built on the **California Housing** dataset.
Each script is self-contained, independently runnable, and designed to teach one concept
at a time вЂ” from raw data ingestion to tracked, registered, and evaluated models.

**Stack:** Python 3.10+ В· scikit-learn В· MLflow 2.9+ В· Pydantic v2

## Data pipeline

Run these scripts in order from the project root. Each one depends on the output of the previous.

```bash
# 1. Download the California Housing dataset from sklearn
python src/data/load_data.py

# 2. Validate the schema of the noisy dataset
python src/data/validate_data.py

# 3. Clean nulls, fix types, remove outliers в†’ data/processed/train_v1.csv
python src/data/preprocess.py

# 4. Create engineered features в†’ data/processed/train_features_v1.csv
python src/data/feature_engineering.py
```

| Script | Input | Output |
|---|---|---|
| `load_data.py` | sklearn API | `data/raw/housing_dataset.csv` |
| `validate_data.py` | `data/raw/housing_noisy.csv` | console summary |
| `preprocess.py` | `data/raw/housing_noisy.csv` | `data/processed/train_v1.csv` |
| `feature_engineering.py` | `data/processed/train_v1.csv` | `data/processed/train_features_v1.csv` |

## MLflow Learning Path

Three scripts that teach MLflow concepts in progression.
Run them in order after the data pipeline is complete.

### Step 1 вЂ” Experiment Tracking

**What you learn:** how to log params, metrics, and model artifacts so every run is
reproducible and comparable.

```bash
python src/models/train.py
```

Trains a `LinearRegression` and a `RandomForestRegressor(n_estimators=100, max_depth=15)`
on `train_features_v1.csv`. Each model is logged as a separate run in the
`california-housing` experiment with metrics `rmse`, `mae`, `r2`.

---

### Step 2 вЂ” Model Registry

**What you learn:** how to promote the best run into a named, versioned model that
consumers can reference without knowing a run ID.

```bash
python src/models/register_model.py
```

Queries all finished runs, picks the one with the lowest RMSE, and registers it as
`california-housing-model` in the MLflow Model Registry.

**Requires:** Step 1 to have been run at least once.

---

### Step 3 вЂ” Model Evaluation

**What you learn:** how to run a formal quality gate on a registered model and promote
it with an alias instead of a stage transition (MLflow 2.9+).

```bash
python src/models/evaluate_model.py
```

Loads the latest version of `california-housing-model`, runs `mlflow.evaluate()` on
the held-out test set, and checks two thresholds:

| Metric | Threshold |
|---|---|
| RMSE | < 0.6 |
| RВІ | > 0.7 |

If both thresholds are met, the alias `champion` is assigned. You can then load the
production-ready model anywhere with:

```python
import mlflow
model = mlflow.pyfunc.load_model("models:/california-housing-model@champion")
```

**Requires:** Step 2 to have been run at least once.
