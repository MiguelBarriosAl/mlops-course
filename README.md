# MLOps Course Project

An educational end-to-end MLOps pipeline built on the **California Housing** dataset.
Each script is self-contained, independently runnable, and designed to teach one concept
at a time — from raw data ingestion to a containerised, served, and CI-tested model.

**Stack:** Python 3.11 · scikit-learn · MLflow 2.9+ · Pydantic v2 · FastAPI · Docker · GitHub Actions

## Data pipeline

Run from the project root. Each script depends on the output of the previous one.

```bash
python src/data/load_data.py          # → data/raw/housing_dataset.csv
python src/data/validate_data.py      # validates schema with Pydantic
python src/data/preprocess.py         # → data/processed/train_v1.csv
python src/data/feature_engineering.py  # → data/processed/train_features_v1.csv
```

| Script | Input | Output |
|---|---|---|
| `load_data.py` | sklearn API | `data/raw/housing_dataset.csv` |
| `validate_data.py` | `data/raw/housing_dataset.csv` | console summary |
| `preprocess.py` | `data/raw/housing_dataset.csv` | `data/processed/train_v1.csv` |
| `feature_engineering.py` | `data/processed/train_v1.csv` | `data/processed/train_features_v1.csv` |

Or run everything at once:

```bash
python src/pipelines/run_data_pipeline.py
```

## MLflow Learning Path

Three scripts that teach MLflow concepts in progression.
Run them in order after the data pipeline is complete.

### Step 1 — Experiment Tracking

**What you learn:** how to log params, metrics, and model artifacts so every run is
reproducible and comparable.

```bash
python src/models/train.py
```

Trains a `LinearRegression` and a `RandomForestRegressor(n_estimators=100, max_depth=15)`
on `train_features_v1.csv`. Each model is logged as a separate run in the
`california-housing` experiment with metrics `rmse`, `mae`, `r2`.

---

### Step 2 — Model Registry

**What you learn:** how to promote the best run into a named, versioned model that
consumers can reference without knowing a run ID.

```bash
python src/models/register_model.py
```

Queries all finished runs, picks the one with the lowest RMSE, and registers it as
`california-housing-model` in the MLflow Model Registry.

**Requires:** Step 1 to have been run at least once.

---

### Step 3 — Model Evaluation

**What you learn:** how to run a formal quality gate on a registered model and promote
it with an alias instead of a stage transition (MLflow 2.9+).

```bash
python src/models/evaluate_model.py
```

Loads the latest version of `california-housing-model`, runs `mlflow.evaluate()` on
the held-out test set, and checks two thresholds:

| Metric | Threshold |
|---|---|
| RMSE | < 0.6 |
| R² | > 0.7 |

If both thresholds are met, the alias `champion` is assigned. You can then load the
production-ready model anywhere with:

```python
import mlflow
model = mlflow.pyfunc.load_model("models:/california-housing-model@champion")
```

**Requires:** Step 2 to have been run at least once.

Or run all three steps at once:

```bash
python src/pipelines/run_training.py
```

## Serving

The API serves predictions from whichever model version holds the `champion` alias.
Run Steps 1–3 above before starting the server.

### Local

```bash
uvicorn src.api.main:app --reload
# Docs: http://localhost:8000/docs
```

### Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Liveness check |
| `GET` | `/model-info` | Name, version, alias, run\_id of the loaded model |
| `POST` | `/predict` | Returns `{"prediction": float, "model_version": str}` |

### Example request

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "MedInc": 3.0, "HouseAge": 25.0, "AveRooms": 5.0, "AveBedrms": 1.0,
    "Population": 1000.0, "AveOccup": 3.0, "Latitude": 37.0, "Longitude": -122.0,
    "rooms_per_household": 5.0, "bedrooms_per_room": 0.2, "log_population": 6.9
  }'
```

## CI/CD

Every push to `master` triggers `.github/workflows/ci.yml`, which runs the full
pipeline in order:

```
load_data → validate → preprocess → feature_engineering
  → train → register_model → evaluate_model
```

If `evaluate_model.py` exits with code 1 (quality gate not met), the workflow fails
and no broken model reaches the registry.
